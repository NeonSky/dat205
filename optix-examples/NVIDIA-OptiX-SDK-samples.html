NVIDIA OptiX SDK Code Samples

The CMake cross-platform build system is used to configure and build the SDK. CMake 2.6.3 or higher must be installed. See platfrom specific build instructions for more details:

Getting Started

The samples below are in alphabetical order. If you're new to OptiX, try starting with the optixTutorial and optixHello samples, or for the Prime API, start with primeSimple.

optixDynamicGeometry 

Compares accel structures for a dynamic scene. A multi level BVH allows faster updates at the cost of slower ray intersections vs. a single level BVH. 		 


Browse Files

optixHello 

This sample demonstrates a minimal OptiX API example. A solid green buffer is drawn. Basic features demonstrated include:
  • Usage of key RT objects; RTbuffer, RTcontext, and RTprogram.
  • Creation, setting and consumption of RTvariables.
  • Simple ray generation program.
  


Browse Files

optixInstancing 

Demonstrates various features related to geometry transformation and instancing. Features:
  • Multiple transforms referencing the same geometry group.
  • Sharing of acceleration structures between multiple groups.
  


Browse Files

optixMDLDisplacement 

Demonstrates how to use MDL materials with vertex displacement.

optixMDLExpressions 

Demonstrates how to use single expressions of MDL materials with OptiX. Features:
  • Using tint and normal expressions of different materials.
  • Using an MDL function as an environment.

optixRaycasting 

This sample uses OptiX as a replacement for Prime, to compute hits only. Compare to primeSimplePP. Shading and ray generation are done with separate CUDA kernels and interop. Also supports an optional mask texture for geometry transparency (the hole in the default fish model).

primeMultiBuffering 

Minimal demonstration of using multi-buffering with Optix Prime. Multi-buffering serves two purposes:
  1. Overlap CPU computation with operations on the GPU
  2. Conserve page-locked host memory. Larger computations can be staged through a smaller amount page-locked memory
